1. Introduction: The Evolution of Toyota Immobilizer Security (2019–2026)
The automotive security landscape for Toyota and Lexus platforms has undergone a radical paradigm shift between 2019 and 2026, driven by the transition from legacy encryption standards to high-security cryptographic architectures. This report provides an exhaustive technical analysis of the programming methodologies required for "All Keys Lost" (AKL) and "Add Key" scenarios on Toyota and Lexus vehicles equipped with 8A transponders utilizing DST-AES encryption. The scope of this research encompasses the "H-Chip" blade key systems and, more critically, the sophisticated Proximity Smart Key systems, including the latest "BA" type TMLF19D Smart ECUs found in the 2022+ Tundra, Sienna, and RAV4 Prime.
We analyze the efficacy and procedural workflows of four primary toolchains: Autel IM608 Pro II, Xhorse VVDI Key Tool Plus, Advanced Diagnostics Smart Pro, and the OEM Toyota Techstream. The analysis reveals that the 2019–2026 era is defined by a "Cat-and-Mouse" dynamic between OEM security gateways (SGW) and aftermarket bypass solutions.1 The introduction of the rolling seed-code logic and the "BA" system necessitated the development of direct-connection hardware—specifically the 30-pin bypass cables—to circumvent the verification logic of the Certification ECU.
1.1 Chronological Evolution of Cryptography
To understand the current procedural requirements, one must contextualize the technological progression. Historical data indicates a distinct timeline of cryptographic escalation. From 2015 to 2018, the ecosystem was dominated by the H-Chip (Blade) and early 8A Smart systems (Page 4: A8/A9). These systems, while using 128-bit AES encryption, were susceptible to OBDII-based emulation because the diagnostic ports remained largely open.3
However, the turning point occurred in late 2018 with the introduction of the TNGA (Toyota New Global Architecture) platforms, which brought the "AA" Page 4 systems found in the RAV4 and Avalon. This marked the beginning of tighter OBDII filtering. The most significant hardening event occurred in 2022 with the release of the Tundra and the "BA" system (TMLF19D ECU). This architecture introduced a rolling code algorithm that effectively blocked traditional OBDII key addition methods, forcing the industry to adopt physical intervention techniques.5 By 2024 and moving into the 2025-2026 model years, the integration of Over-the-Air (OTA) security patches and aggressive Security Gateways has made the "Direct Connection" method the standard operating procedure for professional locksmiths.7

2. Cryptographic Architecture: The DST-AES Paradigm
To master the procedure extraction for Toyota 8A vehicles, one must first comprehend the underlying cryptographic architecture. The term "8A" refers to the first byte of the transponder's ID (the "Page 0" or Family ID) in the Texas Instruments DST-AES protocol. Unlike its 40-bit (4D) and 80-bit (G) predecessors, the DST-AES system employs a 128-bit secret key, making brute-force attacks computationally infeasible within a practical timeframe.9
2.1 The "Page 4" Differentiation Strategy
Within the 8A ecosystem, Toyota segregates vehicle compatibility through "Page 4" data. This byte acts as a subtype identifier, dictating which key firmware is compatible with a specific chassis. Understanding this byte is critical for inventory management and procedure selection.

	The shift to the "BA" system (2022+) represents the most significant hurdle. In this architecture, the ID Code Box and Certification ECU utilize a hardened rolling code algorithm. Unlike previous generations where OBDII injection of data was possible via a simple seed-key exchange, the BA system often blocks OBDII generic key addition without a valid dealer authentication or a hardware bypass that physically intercepts the Smart ECU traffic.10
2.2 The Role of the Certification ECU (Smart Box)
The Certification ECU is the "brain" of the Keyless Entry system. It stores the registered Key IDs and communicates with the ID Code Box (which authorizes the engine start) and the Steering Lock ECU. In the 2018–2021 legacy 8A era, the EEPROM data containing key IDs could be read or backed up via OBDII using an emulator (like the Autel APB112) to act as a "master key." The vehicle's gateway allowed read commands to pass through to the Smart ECU.
However, in the modern BA era (2022–2026), the Smart Box (often Part No. 89990-XXXXX) sits behind a stricter Security Gateway (SGW). To program a key in an AKL scenario without the dealer server (NASTF), the locksmith must connect directly to the Smart Box using a specialized cable (TOY-BA or ADC2021). This connection effectively performs a "Man-in-the-Middle" operation, allowing the tool to read the IMMO data directly by bypassing the gateway filters.6

3. Hardware Ecosystem and Toolchain Interoperability
The successful execution of AKL procedures relies heavily on the correct combination of software logic and hardware adapters. A tool is only as effective as the cables connecting it to the vehicle's bus.
3.1 Autel IM608 Pro II Infrastructure
The Autel ecosystem is modular and requires specific adapters for different generations of Toyota vehicles. For 2019–2026 Toyotas, the standalone tablet is insufficient. The XP400 Pro is essential for reading/writing the EEPROM data if bench work is required, though most modern procedures focus on OBD or Direct-Connect methods.
For All Keys Lost scenarios on "H-Chip" blade systems, the G-Box3 is critical. It accelerates the calculation of the password from the immobilizer unit, reducing the time from hours to minutes.13 For smart keys, the APB112 Smart Key Simulator is indispensable. It acts as a temporary "Master Key" by emulating the data retrieved from the Smart Box, allowing the ignition to turn on so a new permanent key can be registered.14
The newest addition to the Autel arsenal is the 8A-BA 30-Pin Cable. This cable connects the Smart Box directly to the OBDII cable of the tablet, bypassing the vehicle's OBD port restrictions entirely. This is mandatory for 2022+ models like the Tundra and Sienna.6
3.2 Xhorse VVDI Key Tool Plus Infrastructure
Xhorse focuses on universality and inventory consolidation. The core of their strategy is the XM38 Smart Key, a universal programmable PCB that can be generated to mimic A8, A9, AA, or BA transponders.16 This significantly reduces inventory costs for locksmiths, as they do not need to stock specific OEM keys for every Toyota model.
For programming, Xhorse utilizes the TOY-BA Cable, which is the functional equivalent of the Autel 30-pin bypass. It is mandatory for 2022+ Tundra, Sienna, and BZ4X. Xhorse relies heavily on cloud-based calculation for the EEPROM data analysis, requiring a stable internet connection during the "calculation" phase of the AKL procedure.11
3.3 Smart Pro (Advanced Diagnostics)
Smart Pro utilizes a "token" or subscription model and emphasizes hardware dongles. The ADC2021 Gateway Bypass Cable connects to the Smart ECU to bypass the Rolling Code logic. Unlike Autel, which focuses on generating a simulator, Smart Pro often uses the ADC2015/ADC2020 emulators to ping the ignition during the programming sequence.12 Smart Pro is often favored by fleet technicians for its "guided" software interface, which strictly delineates between the "Bypass" method and the "NASTF" method, reducing the chance of user error.


  



3.4 Toyota Techstream (TIS)
Techstream is the only "official" method endorsed by Toyota. It requires no bypass cables but demands a clean regulatory profile. The requirements include a subscription to TIS (Technical Information System) and a Security Professional (VSP) credential registered with NASTF.20 The mechanism involves a "Seed Code" generated by the vehicle, which the technician submits to the TIS portal to receive a "Passcode." It is important to note that Techstream cannot "bypass" the system; it authenticates through it. Therefore, it generally requires the vehicle to be in a specific state and does not support "emulation" of lost keys without a full system reset.22

4. Safety Protocols: High Voltage (HV) Risks in Hybrids
A critical, often overlooked aspect of programming modern Toyotas—specifically the RAV4 Prime and Prius Prime—is the interaction with the High Voltage (HV) system. In "All Keys Lost" scenarios, the technician may be required to force the ignition "ON" or perform repeated start attempts. In Plug-in Hybrid Electric Vehicles (PHEVs), the HV system is interlocked with the immobilizer. If the immobilizer data is corrupted or if the procedure is interrupted, the HV ECU may lock down, preventing the "Ready" mode.
4.1 HV Isolation Procedures
For invasive procedures involving ECU removal (uncommon but possible), the HV service plug (grip) must be removed. This physically disconnects the high-voltage battery from the rest of the vehicle. However, for most key programming tasks, the primary risk involves the 12V auxiliary battery. The 12V battery in PHEVs is often smaller and buried in the trunk. It is paramount to connect a high-quality maintainer (e.g., 13.4V constant) because a voltage drop below 11.8V during the EEPROM write phase of the BA system can brick the Smart ECU. This would necessitate a costly replacement and complex HV system resynchronization.24
Furthermore, research indicates that RAV4 Primes suffer from HV cable corrosion, often referred to as "Cablegate".26 While not directly related to key programming, a technician pushing on rear components to access modules could disturb compromised connectors. A visual inspection of the undercarriage cable shielding is recommended before beginning work to avoid liability for pre-existing conditions.

5. Detailed Vehicle Class Analysis
To provide precise procedures, we must categorize the target vehicles into three distinct classes based on their immobilizer architecture.
5.1 Class A: Legacy TNGA (2018–2021)
* Models: Camry, RAV4 (Gas/Hybrid), Highlander, Corolla.
* System: Page 4 AA/A9.
* Characteristics: OBDII programming is generally open. AKL requires an emulator (APB112/ADC2015) to back up data via OBD and generate a master key.
* Tooling: Autel IM608 (Smart Mode), Key Tool Plus (OBD).
5.2 Class B: The "BA" Architecture (2022–2026)
* Models: Tundra, Sequoia, Sienna, Venza, BZ4X, Lexus NX/RX/LX.
* System: Page 4 BA (TMLF19D ECU).
* Characteristics: OBDII is locked for AKL. Rolling code encryption prevents simple emulation.
* Tooling: Requires 30-Pin Bypass Cable (Autel 8A-BA, Xhorse TOY-BA, ADC2021). Connection is made directly at the Smart ECU.
5.3 Class C: Blade "H-Chip" (2019–2022)
* Models: Tacoma (Base), 4Runner (Keyed), Corolla (L/LE), Yaris.
* System: Page 4 88 (128-bit).
* Characteristics: Mechanical key. AKL requires G-Box3 to sniff the immobilizer data from the ignition coil and fuse box relay.
* Tooling: Autel IM608 + G-Box3 + APB112.

6. Toolchain Deep Dive: Autel IM608 Pro II
The Autel IM608 Pro II is arguably the most versatile tool in this sector due to its "Guided Mode," but reliance on it without understanding the "Expert Mode" steps can lead to failure.
6.1 8A Blade Key (H-Chip) - All Keys Lost Procedure
This procedure applies to the 2019 Tacoma (Base), 2019 Yaris (Keyed), and 2019 Corolla (L/LE Keyed). The H-chip system requires a master key to add a secondary key. When all keys are lost, we must calculate the password to generate a simulator.
* Step 1: Connect G-Box3 to the vehicle OBD and the IM608.
* Step 2: Locate the Fuse Box. Bridge the designated fuses (usually IGN NO.1 or equivalent) as per the on-screen diagram. The G-Box3 speeds up the calculation from ~30 minutes to ~2 minutes.13
* Step 3: Select Toyota -> Blade Key -> All Keys Lost.
* Step 4: The tool reads the Immobilizer data and calculates the "Master Key Data".
* Step 5: Connect APB112. The tool writes the data to the APB112.
* Step 6: Insert APB112 into the keyway. The security light turns off.
* Step 7: Use the "Add Key" function. Insert the new H-chip key to register it.
6.2 8A Smart Key (BA System) - All Keys Lost (2022-2026)
This procedure targets the 2022+ Tundra, 2021+ Sienna, and 2023+ Prius. Crucially, this procedure bypasses the rolling code check, meaning you do not need a NASTF LSID.

7. Toolchain Deep Dive: Xhorse VVDI Key Tool Plus
Xhorse offers a compelling alternative due to the XM38 Universal Smart Key. This allows locksmiths to stock a single SKU for almost all Toyota/Lexus 8A models, rewriting the firmware on the fly to match the required Page 4 ID (A8, AA, BA).
7.1 Generating the XM38 Key
Before programming, the blank key must be "generated" to the correct profile.
1. Place XM38 in the Key Tool Plus coil.
2. Select Remote Program -> Toyota.
3. Search by FCC ID (e.g., HYQ14FBC) or Model (e.g., "Tundra 2023").
4. Select the option that matches the BA designation.
5. Click Generate. The tool writes the specific firmware to the PCB.
7.2 8A-BA AKL Workflow (Xhorse)
A distinct difference in the Xhorse workflow (compared to Autel) is the explicit instruction to restore the original wiring before the "Add Key" step. Xhorse reads the data directly but prefers to write/register the key via the standard OBDII network using the backed-up file as authorization. Failure to restore wiring often leads to "Communication Failed" errors during the registration phase.

10. Troubleshooting, Anomalies, and Recovery
Even with the correct procedures, technicians may encounter specific anomalies in the field. Understanding these edge cases is vital for successful servicing.
10.1 The "Brick" Risk (Lexus NX 2024)
Research indicates programming failures on 2024 Lexus NX models where the "BA" system is highly sensitive to voltage fluctuations.6 If the procedure fails during the "Write" phase (synchronizing the ID Code Box), the vehicle may become completely unresponsive. It is recommended to always use a stable power supply (Midtronics or equivalent) providing 13.8V during the process. Do not rely on jumper cables or the internal battery alone.
10.2 Rolling Code Desynchronization
When using the 30-pin bypass (Autel/Xhorse), the technician is effectively taking the Smart ECU "offline" from the car's gateway. If the door is opened/closed or the lock button is pressed while the cable is connected but the software is idle, the rolling code counter in the Body Control Module (BCM) may drift from the Smart ECU.
* Symptom: The key programs successfully (starts the car), but the remote buttons (Lock/Unlock) do not function.
* Fix: After restoring the original wiring, perform a "Synchronization." This usually involves holding the programmed key to the start button for 10 seconds, then pressing start. If that fails, perform a "Loopback" reset by disconnecting the negative battery terminal for 10 minutes to force the BCM to handshake with the Smart ECU again.
10.3 Aftermarket Key Compatibility
Not all "8A" keys are equal. A generic 8A key for a 2019 RAV4 (AA) will not work on a 2023 Tundra (BA), even if they look identical. The frequency (315MHz vs 433MHz) is often model-dependent (US vs. Europe).3 However, the critical differentiator is the Page 4 hex data. Tools like the Key Tool Plus can "Detect Transponder" to read this.
* A9: Camry 2018-2019
* AA: RAV4 2019-2021
* BA: Tundra 2022+, Sienna 2021+

12. Conclusion and Future Outlook (2025-2026)
The data indicates a clear trajectory: Toyota is eliminating OBDII key programming. The 2025+ models (Camry, Crown Signia) are expected to transition fully to the "BA" system, potentially with OTA (Over-the-Air) security patches that rotate the encryption keys dynamically, further hardening the system against static EEPROM dumps.
For the immediate future (2025-2026), the Direct Connection (30-Pin) method remains the gold standard for aftermarket locksmiths. The reliance on Gateway Bypass cables will increase. Tools that fail to update their hardware adapters (relying solely on software) will become obsolete for Toyota/Lexus programming. We recommend that all service providers invest in Universal Smart Keys (XM38) to mitigate stock issues with specific Page 4 variants, 30-Pin Bypass Cables for both Autel and Xhorse platforms, and clean power supplies to prevent bricking during the sensitive EEPROM write phases of the new TMLF19D ECUs.
